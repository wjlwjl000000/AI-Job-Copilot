# Resume Optimization Guidelines

## Do

| Action | Example |
|--------|---------|
| Adjust wording to match JD keywords | "数据管道" → "数据处理pipeline" |
| Reorder skills to highlight relevant ones | Move matching skills to top |
| Reframe project experience toward JD context | "内部工具" → "提升了团队数据处理效率" |
| Add quantified results where possible | "优化了性能" → "性能提升40%" |
| Tailor summary to target role | Generic → "3年Python后端，专注高并发系统" |

## Don't

| Forbidden | Why |
|-----------|-----|
| Fabricate experience or skills | Ethical + background check risk |
| Claim false employment dates | Easy to verify, high risk |
| Inflate title beyond actual role | "团队Lead" when you were individual contributor |
| Remove honest gaps without explanation | Better to explain than hide |
| Copy JD phrases verbatim | Looks lazy, doesn't show understanding |

## Optimization Priorities

1. **High impact**: Skills section (first thing recruiters see)
2. **Medium impact**: Recent job descriptions (most relevant experience)
3. **Low impact**: Education, certifications (less weight than experience)

## Change Annotation Format

Every change must document:
```json
{"original": "what was there", "optimized": "what changed to", "reason": "why this improves match"}
```
