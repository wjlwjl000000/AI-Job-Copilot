# Interview Question Types

## 1. Behavioral (行为面试)
**Purpose**: Assess past behavior as predictor of future performance.

**JD Patterns**: "团队协作" "沟通能力" "抗压能力" "ownership"

**Example Questions**:
- "Describe a project where you had to collaborate across teams."
- "Tell me about a time you resolved a technical disagreement."
- "How do you prioritize when facing multiple deadlines?"

## 2. Technical Deep Dive (技术深挖)
**Purpose**: Verify claimed technical expertise.

**JD Patterns**: Specific tech stack mentions, "精通" "深入研究"

**Example Questions**:
- "Walk me through how you'd design a rate limiter for a distributed system."
- "What's the difference between async/await and threading in Python?"
- "Explain how you'd debug a memory leak in production."

## 3. System Design (系统设计)
**Purpose**: Assess architecture thinking.

**JD Patterns**: "架构设计" "高并发" "分布式" "微服务"

**Example Questions**:
- "Design a job matching system that handles 10M profiles and 1M JDs."
- "How would you build a real-time notification system?"
- "Design a resume parser that handles multiple formats."

## 4. Project Deep Dive (项目深挖)
**Purpose**: Understand actual contribution depth.

**JD Patterns**: Any project experience listed.

**Example Questions**:
- "What was the hardest technical challenge in [project X]?"
- "If you could rebuild [project Y] from scratch, what would you change?"
- "How did you measure the success of [project Z]?"

## Question Generation Rules
- Mix types: 1 behavioral + 2-3 technical + 0-1 system design
- Target weaknesses first: if profile lacks distributed experience, emphasize system design
- Each question must have: focus area (what it tests) + tips (what a good answer covers)
