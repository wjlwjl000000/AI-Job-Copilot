本轮改进建议汇总

1. AI API 配置

使用 ChatOpenAI()：model="gpt-5-mini", base_url="https://api.chatanywhere.tech"

2. 向量存储

应用启动时即连接 Chroma，用 add_documents() 一行写入，不手写向量字段。

3. 上下文管理

自定义 middleware：滑动窗口 + 自动摘要。窗口内保留原文，窗口外压缩为摘要。按字符数控制（非 token）。不适用 LangChain 内置 SummarizationMiddleware。

4. 系统流程软连接

Planner 只规划用户当前明确请求的任务。Synthesizer 在回复末尾附上"基于当前状态，您可以..."的软性建议。用户可自由选择任何能力，无强制流程链路。Agent 之间的连接 = 用户的选择权。

5. Supervisor 任务依赖解析

Planner 需识别用户消息中的先后顺序："先优化简历，再准备面试" → Step1: optimize → Step2: interview（依赖 Step1）。Executor 按依赖层执行。

6. Agent Card 按 A2A 标准注册

Agent Card 端点：/.well-known/agent-card.json。Supervisor 启动时广播/轮询发现。前端提供注册界面，用户输入 Agent URL → 拉取 Card → 持久化。

7. 系统 Agent 不可删除

内置 5 个 Agent 是系统骨架，不可删除。用户可通过前端注册自定义 Agent，可管理。

8. 通信协议按 A2A 标准

废弃自定义 JSON 格式。改用 A2A 标准 JSON-RPC 2.0：

  • 方法: tasks/sendMessage
  • Message: {role, parts: [{type, text/content/href}]}
  • Task 状态: submitted → working → input-required → completed / failed / canceled
  • input-required = 当前的 "priority=true"
  • SSE 事件: messageDelta, taskStatusUpdate, taskArtifactUpdate

9. System Prompt SKILL 占位符

不再写静态 system_prompt。使用占位符：

"你是简历与画像专家。\n可用技能：{skills_list}\n当前技能详情：{skill_content}"

SkillLoadingMiddleware 在加载 SKILL 时填充。

10. Supervisor 异步流式输出

Synthesizer 内部使用 LLM stream() 逐 chunk 输出自然语言，包装在 graph.astream() 事件流中。前端收到增量文本逐字渲染。

11. Support Agent 永不 priority=true

Support 的 A2A 响应永远 state: "completed"。结果总是嵌入调用者（Matching/Profile/Interview）的返回值。不会独立触发中断。

12. 用户回复传入：Command(resume=...)

Executor 内 interrupt() 的返回值就是用户回复，不需要额外 state 字段：

user_answer = interrupt({"type": "user_question", ...})
# user_answer 直接可用，发给原 Agent 继续

恢复方式：graph.astream(Command(resume={...}), config)
