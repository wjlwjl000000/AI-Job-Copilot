数据模型 & API 设计

核心数据实体

UserProfile (用户求职画像) skill_tags: jsonb          # 技能标签+等级 work_years: int             # 工作年限 education: jsonb            # 学历/学校/专业 projects: jsonb              # 项目经验 target: jsonb                # 目标{城市,薪资,行业,岗位} preference: jsonb            # 偏好设置 embedding: vector(1536)      # 画像向量 scores: jsonb                # 竞争力/匹配度/完整度 Resume (简历) user_id → User base_version: bool          # 是否基础版本 target_role: str             # 目标岗位(版本标签) content: jsonb               # 结构化简历内容 file_path: str              # 原始文件路径 match_score: float          # 匹配度评分 Job (职位) source: str                  # 来源(BOSS/拉勾/手动) jd_content: text            # 职位描述 requirements: jsonb          # 结构化需求 embedding: vector(1536)      # JD向量 status: str                 # 招聘中/已关闭 Application (投递记录) user_id → User   resume_id → Resume   job_id → Job status: enum                 # 待投/已投/初筛/面试/offer/拒 timeline: jsonb              # 状态变更时间线 notes: text                  # 用户备注 Interview (面试记录) application_id → Application questions: jsonb            # 面试问题+回答 feedback: text              # AI评估反馈 weaknesses: jsonb            # 薄弱项+改进建议 ExperienceStory (经历故事) embedding: vector(1536)      # 内容向量 tags: jsonb                  # 行业/岗位/年限/结果 content: text                # 故事内容 source: str                  # 来源(爬取/用户贡献)

核心 API

POST /api/profile/analyze          # 解析简历→生成画像 GET  /api/profile                 # 获取画像 PUT  /api/profile                 # 更新画像 POST /api/resumes/upload          # 上传基础简历 POST /api/resumes/optimize        # 为目标JD优化简历(生成新版本) POST /api/resumes/{id}/score      # 简历-JD匹配度评分 POST /api/jobs/search              # 搜索/匹配职位 GET  /api/jobs/{id}               # 职位详情 POST /api/applications              # 创建投递记录 GET  /api/applications              # 投递列表(看板) POST /api/interviews/generate      # 生成面试问题 POST /api/interviews/evaluate      # 评估回答 POST /api/support/match-story      # 匹配相似经历故事 GET  /api/support/feed            # 情感支持内容流

RAG 情感支持流程

1. 触发节点：匹配度低 / 投递被拒 / 面试失败 / 拿到offer
2. 画像向量检索 → pgvector 中查找与用户画像相似的经历故事
3. Agent 生成 → 基于检索结果 + 用户当前处境 → 个性化鼓励文案
4. 前端展示 → 温和推送，不做弹窗打扰
